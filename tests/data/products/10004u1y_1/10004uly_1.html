Preliminary Finite Fault Results for the Mar 02, 2016 Mw 7.8 -4.9050, 94.2360 Earthquake (Version 1)

DATA Process and Inversion

We used GSN broadband waveforms downloaded from the NEIC waveform server. We analyzed 50 teleseismic broadband P waveforms, 17 broadband SH waveforms, and 72 long period surface waves selected based on data quality and azimuthal distribution. Waveforms are first converted to displacement by removing the instrument response and are then used to constrain the slip history using a finite fault inverse algorithm (Ji et al., 2002). We begin modeling using a hypocenter matching or adjusted slightly from the initial NEIC solution (Lon. = 94.236 deg.; Lat. = -4.905 deg., Dep. = 24.0 km), and a fault plane defined using either the rapid W-Phase moment tensor (for near-real time solutions), or the gCMT moment tensor (for historic solutions).

Result

After comparing waveform fits based on the two planes of the input moment tensor, we find that the nodal plane (strike= 274.0 deg., dip= 84.0 deg.) fits the data better. The seismic moment release based upon this plane is 6.0e+27 dyne.cm (Mw = 7.8) using a 1D crustal model interpolated from CRUST2.0 (Bassin et al., 2000).
